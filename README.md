#### ✨ Funcionalidades Principales
```
✓ Analizador de mensajes en tiempo real
✓ Detección de palabras clave fraudulentas
✓ Pipeline de procesamiento NLP animado
✓ Bandeja de mensajes pre-analizados
✓ Filtrado por nivel de riesgo
✓ Estadísticas y métricas del modelo
✓ Visualización de arquitectura del sistema
✓ Diseño responsive (móvil + desktop)
```

#### 🛠️ Stack Tecnológico

### 2. 🏫 **Portal Estudiantil — UE Los Andes**

**Sistema Web de Gestión Educativa**

Portal estudiantil profesional para la Unidad Educativa Los Andes. Aplicación web con autenticación segura, panel administrativo y gestión de estudiantes. Backend PHP con PostgreSQL, desplegado como serverless en Vercel.

#### 🔗 Enlaces del Proyecto
- **Repositorio GitHub:** [`ue-los-andes`](https://github.com/eatechristianarielcondoriqu-dot/Grupo2-Ing-Sof/tree/main/ue-los-andes)
- **Sitio en Vivo:** [ue-los-andess.vercel.app](https://ue-los-andess.vercel.app/)

#### ✨ Funcionalidades Principales
```
✓ Autenticación segura con login
✓ Panel administrativo protegido
✓ Tabla de gestión de estudiantes
✓ Cookies firmadas con HMAC-SHA256
✓ Rutas limpias sin .php
✓ Base de datos PostgreSQL en Neon
✓ Despliegue serverless en Vercel
✓ Seguridad: password_hash + PDO
```

#### 🛠️ Stack Tecnológico
- **Backend:** PHP 8.1+
- **Base de Datos:** PostgreSQL (Neon)
- **Hosting:** Vercel (PHP Runtime)
- **Autenticación:** Cookie firmada HMAC
- **Seguridad:** PDO con parámetros preparados
- **Metodología:** Scrum

✓ docs(README): instrucciones de despliegue

## 🔄 Filosofía de Commits

Ambos proyectos siguen el **formato convencional de commits**:

```
<tipo>(<componente>): <descripción>
```

### Tipos utilizados:
- **feat:** Nueva funcionalidad
- **fix:** Corrección de bug
- **docs:** Cambios en documentación
- **refactor:** Reorganizar código
- **style:** Formato/estilos
- **test:** Agregar tests

### Ejemplos:
```bash
git commit -m "feat(auth): agregar login seguro"
git commit -m "fix(ui): corregir responsive design"
git commit -m "docs(README): agregar instrucciones"
git commit -m "refactor(api): optimizar consultas"
```

Esto permite:
- ✅ Ver el historial de desarrollo
- ✅ Entender cambios por tipo
- ✅ Evaluar la evolución del proyecto
- ✅ Demostrar workflow profesional
